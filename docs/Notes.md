## 系統架構概要

本系統預計分為三個權限端：**學生端**、**老師端**以及**管理員端** 。

---

學生端功能需求 

### 1. 登入與平台

* 
**登入方式**：支援 Google 帳號登入或註冊，便於管理與後續追蹤 。


* 
**支援平台**：需涵蓋電腦端與手機端（.html 或 App 形式），考量到實際考試為上機考，但練習時使用者傾向使用手機 。



2. 測驗科目選擇 

目前規劃包含醫檢師六大科目（未來可擴充）：

1. 臨床生理學與病理學 


2. 臨床血液學與血庫學 


3. 醫學分子檢驗學與臨床鏡檢學 


4. 微生物學與臨床微生物學 


5. 生物化學與臨床生化學 


6. 臨床血清免疫學與臨床病毒學 備註：一年有兩次考試，故每年共計 12 份考古題 。



### 3. 測驗模式設定

* 
**題數選擇**：可設定 5、10 題或完整的 80 題（國考單科標準題數） 。


* 
**出題邏輯**：單份完整考古題、單份隨機、多份隨機、選項隨機 。


* 
**AI 難易度分級**：導入 AI 判讀，將題目分為簡單、中等、困難模式 。


* 
**計時功能**：提供不計時、計時模式，並記錄單題用時與剩餘總時間 。



### 4. 輔助與詳解功能

* 
**紀錄設定**：可勾選是否將該次測驗結果納入歷史紀錄 。


* 
**提示系統**：題目中可點選工具提示（需於測驗前勾選是否開啟 AI 提示按鈕） 。


* 
**詳解模式**：可選擇「即時知道解答與 AI/網路詳解」，或「模擬考模式」於交卷後統一查看 。



---

考試介面 UI 需求 

* 
**呈現方式**：一次呈現一題 。


* 
**題目資訊**：標註來源（例如：113 年第一次臨床生理第 3 題）與題號 。


* 
**作答功能**：選項前提供「刪除（刪去法）」功能 。


* 
**進度顯示**：顯示目前答題進度條（如：20/80） 。


* 
**導覽按鈕**：上、下一題跳轉按鈕，以及提早交卷按鈕 。


* 
**匯出功能**：支援將試卷或作答紀錄匯出成 PDF 。



---

數據分析與 AI 回饋 

* 
**答題回饋**：顯示答對率，並由 AI 判讀作答者的弱點與盲區 。


* 
**強化練習**：AI 根據分析結果，在練習模式中增加弱點類型的出題頻率 。


* 
**成長追蹤**：AI 比較歷史紀錄，判斷測驗表現是否退步，並提出改善方案與讀書方法 。


* 
**時間效率**：AI 比較「題目應耗時」與「實際用時」，給予作答速度建議 。


* 
**存檔功能**：作答紀錄、詳解、分析報告與作答日期皆存於個人資料中 。



---

老師端與管理員端 

### 老師端

* 
**班級管理**：透過課程代碼或邀請連結，讓學生自動加入班級 。


* 
**進度監控**：可查看班級所有學生的作題情況 。



### 管理員端

* 
**權限管理**：具備全權限，管理所有人員 。


* 
**數據調閱**：可監控所有人進度並調閱紀錄 。


* 
**題庫維護**：負責不斷更新題庫內容 。



---

擴充功能 

* 
**AI 預測模式**：根據題庫題目方向與格式，自動生成一份模擬試卷 。

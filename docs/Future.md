## 老師端（Phase 6）

### 角色說明
老師帳號由管理員指派，或透過特定註冊流程取得。登入後進入老師專屬介面。

---

### 頁面流程

```
登入頁（同學生端）
  └─→ 老師主選單
        ├─→ 班級管理頁
        │     ├─→ 建立班級
        │     ├─→ 班級詳細頁 ─→ 學生個人進度頁
        │     └─→ 邀請學生（產生課程代碼 / 邀請連結）
        └─→ 全班統計頁
```

---

### 各頁面規格

#### 1. 老師主選單（Teacher Home）
- 顯示所有管理中的班級列表
- 「建立新班級」按鈕
- 登出按鈕

---

#### 2. 班級管理頁（Class Management）
- 班級名稱、建立日期
- 學生人數
- 產生 **課程代碼**（6位英數）或 **邀請連結**，學生輸入後自動加入班級
- 學生名單列表：
  - 顯示每位學生姓名、最近作答日期、累計題數
  - 點擊 → 學生個人進度頁

---

#### 3. 學生個人進度頁（Student Progress）
- 該學生的歷史紀錄（日期、科目、答對率）
- 各科答對率統計圖
- AI 分析摘要（若 Phase 5 已完成）

---

#### 4. 全班統計頁（Class Statistics）
- 全班各科平均答對率
- 答題人數 / 總學生數（參與率）
- 各題錯誤率排行（哪些題目全班最多人答錯）
- 可依科目篩選

---

### 資料庫擴充需求
- `Class` 表：班級 id、名稱、老師 id、課程代碼
- `ClassMember` 表：班級 id、學生 id、加入日期
- User 表新增 `role` 欄位：`student` / `teacher` / `admin`

---

### 與學生端的整合點
- 學生加入班級後，其測驗紀錄（已勾選「納入歷史紀錄」者）自動對老師可見
- 學生端介面不需改動，僅後端 API 新增查詢權限控制

---
